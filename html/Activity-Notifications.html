----Activity Notifications----

Now with Authentication!

Using sockets, this page shows real time activity notifications for the authenticated POST, PUT, and DELETE operations.

Week 8 assignment update:
1st API: Notifications have been upgraded to use the Notification API with a fallback to alert(). This means users must allow browser notifications to receive real-time updates. If browser notifications are blocked, the system will use alert() as a fallback. Testing this works best on Edge or Firefox by executing the get, post, put, or delete operations on a separate device and observing the notifications on a laptop/desktop computer.
2nd API: The Web Storage API is being used to persist user theme preferences (background color, font type, and font color) across sessions. When a user selects their preferred theme settings, these preferences are saved in the browser's local storage. Upon revisiting the page, the stored preferences are retrieved and applied.
Please see GitHub linked in assignment description for more details.
